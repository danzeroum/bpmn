Event Subprocess — o contêiner que fecha a aproximação honesta da E-4/E-5

Hoje o start de erro "dentro de subProcess" é uma aproximação declarada — o contêiner real da OMG (subProcess triggeredByEvent="true", borda pontilhada, start tipado único, interrupting/não-interrupting) não existe. Este handoff o traz por inteiro: modelo OMG puro, visual normativo, veto de fluxo de sequência declarado, lint EVT_SUBPROC_*, e o disparo honesto no simulador estendendo o matching da E-6. Pré-requisito: Handoff 16 completo na main (está).

Regra estrutural no core: fluxo de sequência de/para o contêiner é vetado pela regra padrão (canal do veto, mensagem nomeando a regra OMG) — e o contêiner é isento do unreachable (molde boundary).

GESTO DE CONEXÃO NO CONTÊINER

4e Disparo honesto no simulador — estende o matching da E-6 precedência OMG documentada · interrupção nomeada na trilha
CARD «LANÇAR ERRO» — CANDIDATOS POR ESCOPO
err-limite → event subprocess «Tratar exceções» MESMO ESCOPO — VENCE
err-limite → boundary em «Subprocesso S» escopo externo — preterido
Erro não catalogado → catch-all (reforço 10 da E-6, mantido)
TRILHA:
✓ caught by event subprocess «Tratar exceções» (errorRef "err-limite")
⛔ interrupting: 2 tokens cancelados em «Subprocesso S»
→ token no start st-1 (não-interrupting: escopo segue)
SEMÂNTICA VINCULANTE
Candidatos de throwError/Signal/Message passam a incluir starts de event subprocess do ESCOPO do token
Precedência OMG documentada: event subprocess do mesmo escopo VENCE boundary do escopo externo — não é ambiguidade (teste vinculante); empate no mesmo nível = BlockedDecision nomeando candidatos
Interrupting: cancela TODOS os tokens do escopo hospedeiro — trilha nomeia a contagem e o escopo · não-interrupting: token paralelo, escopo segue
Timer/conditional start: NUNCA auto-dispara — card de decisão manual (molde boundaryOptions), declarado
Propagação além do escopo direto segue NÃO simulada — limitations.md atualizado (remove "event subprocess", mantém propagação/escalation/compensação)
